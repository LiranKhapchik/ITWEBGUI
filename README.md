# פורטל מחשוב ושליטה - מרכז שליטה צוות מחשוב

פורטל אינטרנט פנימי לניהול מרכז השליטה של צוות המחשוב. המערכת מרכזת את כלל הפעילויות של צוות המחשוב, לרבות פתיחה ומעקב אחר תקלות, ניהול מלאי סיריאלי מקיף, השאלת ציוד, שריון כיתות וחדרי דיונים, ניהול משמרות ותורנויות של הצוות, והפקת דוחות רשמיים בפורמט צבאי.

---

## 🚀 טכנולוגיות וארכיטקטורה

המערכת בנויה בארכיטקטורת Client-Server קלה ויעילה המותאמת להרצה בסביבות סגורות ומאובטחות:
- **צד לקוח (Client Side):**
  - **HTML5 & Vanilla JavaScript (ES6):** ניהול מצב (State Management) מקומי וסנכרון מול השרת.
  - **Tailwind CSS:** מיושם באמצעות ספריית JS מקומית לתמיכה מהירה בעיצוב מודרני מותאם אישית (כולל Dark Mode מלא).
  - **SheetJS (xlsx.full.min.js):** תמיכה מלאה בייצוא/ייבוא מדריכי תקלות מובנים מקובצי Excel (.xlsx).
- **צד שרת (Server Side):**
  - **PHP 8.1:** מנוע ה-REST API לניהול כלל בקשות המידע וסנכרון המצב.
  - **קובצי JSON:** שמירת הנתונים בקובצי JSON מקומיים תחת תיקיית `data/` המותאמים להרצה אופליין מלאה וקלה.
- **סביבת הרצה ופריסה:**
  - **Docker:** הרצה מבוססת אימג' Red Hat UBI 9 PHP 8.1, המותאם לעבודה כמשתמש שאינו root.
  - **OpenShift:** הגדרות מובנות לפריסה בקלאסטר הכוללות Persistent Volume Claim (PVC) לשמירת קובצי הנתונים, Service ו-Route מאובטח.

## 🛠️ הוראות הרצה ופריסה

### הרצה מקומית באמצעות Docker

ניתן להריץ את האתר בשתי דרכים מקומיות:

#### דרך א': באמצעות Docker Compose (מומלץ - שומר על הנתונים תיקיית data מקומית)
הרצו מתיקיית השורש:
```bash
docker compose up -d
```
האתר יהיה זמין בכתובת: `http://localhost:8080` והנתונים יישמרו בתיקיית `data` המקומית בפרויקט.

#### דרך ב': באמצעות פקודות Docker ישירות
1. **בניית ה-Image:**
   ```bash
   docker build -f docker/Dockerfile -t military-portal:latest .
   ```
2. **הרצת ה-Container:**
   ```bash
   docker run -d -p 8080:8080 --name military-portal-app military-portal:latest
   ```
   המערכת תהיה זמינה בדפדפן בכתובת: `http://localhost:8080`

### פריסה ב-OpenShift
הקובץ [openshift-deployment.yaml](file:///c:/Users/User/Desktop/ATAR%20MCHSUV/אתר/docker/openshift-deployment.yaml) כולל את כל הגדרות הפריסה הרשמיות. כדי לפרוס את המערכת בקלאסטר, הרץ:
```bash
oc apply -f docker/openshift-deployment.yaml
```
*הערה: התיקייה `/opt/app-root/src/data` ממופה ל-PVC בנפח 1Gi על מנת לשמור על קובצי ה-JSON קבועים ומוגנים מפני מחיקת פודים.*
